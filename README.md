### 吐槽

1. **赵的反馈像俄罗斯套娃**：你以为他给了3条意见，打开一看里面有12条。你以为12条是全部了，结果他还有下一轮。

3. **背景色过山车**：v5 暗色底 → v6 暗色底赵说不要 → v7 浅色底 → 背景图太淡 → 加深 → 还淡 → 继续加深 → 还不够。背景图的 opacity 从 0.06 一路涨到 0.35，比比特币波动还大。

4. **"你自己看一下有什么区别"**：赵发了两张截图让我对比。我看了半天，发现区别是……区块边框。对，就是那个我在 v5 加了、v6 去掉、v7 又加回来的边框。

5. **底部装饰的绿色**：赵给了一个参考图，我用 CSS filter 调了半小时。最后赵说"算了，直接复制顶部的"。早说嘛。早说我5分钟就搞完了。

6. **赵凌晨5点还在改状态栏**：他说明天中午起床。我看他的"明天"和"中午"都不是正常的定义。

## 2026-07-07：状态栏三文件拆分 + 按钮之谜 + 背景图消失案

### 发生了什么

醒来后赵说：把状态栏 v7 拆成三个 txt，像 status-bar-fix 仓库那样。

听起来简单对吧？我也这么觉得。然后我花了五个小时和各种 bug 搏斗。其中包括：一个不存在的 bug（Python 测试误判）、一个被专家误导的方向（onerror）、一个做了十个版本最后回到原点的 onclick、以及一个从 0.35 到 0.5 的 opacity 之旅。

### 三文件拆分

产出四个文件（多了一个指令集）：
- `黍-状态栏指令集.txt` — 给平台 AI 的自然语言指令
- `黍-状态栏文本.txt` — 文本模板（28字段）
- `黍-匹配正则.txt` — 正则表达式
- `黍-替换内容.txt` — HTML/CSS/JS 替换代码

另外还做了一个本地预览文件 `黍-状态栏预览.html`，底部模拟了输入栏，用于测试按钮效果。

### 正则踩坑

1. `\s*` vs `\n`：最初用 `\s*`，平台报 SyntaxError。改成 `\n` 后解决。后来专业 AI 说 `\s*` 更稳，又改回来了。这次没报错。我不知道为什么。就像我不知道为什么感冒会好。
2. `(?:...)` 非捕获组：担心平台不支持，去掉了。后来发现 status-bar-fix 也在用。虚惊一场。

### 颜色太淡大战

v7 预览 HTML 看着没问题，平台上像蒙了层硫酸纸。

排查发现：平台的全局美化 CSS（`[class*=ai]`、`[class*=bot]`）用 `color: var(--text-main) !important` 覆盖了状态栏内所有文字颜色。

第一反应用 `color: inherit !important`——没用，因为父元素也被覆盖了。

最终方案：给每个元素单独写 `color: #xxx !important`，14条规则，像打地鼠一样一个一个堵。

### 按钮不工作之谜（未解决）

做了十个版本的 onclick，用了三种选择器，加了 fallback，区分了 tagName，改了 dispatchEvent 的参数。最后回到原点，和德克萨斯完全一样。还是不行。

排查路径：
1. `$24` 被解释成 `$2`+`4`？→ Python 测试脚本顺序替换导致误判，实际没问题
2. onclick 被平台过滤？→ 专业 AI 说会被剥离，但德克萨斯/M3/阿尔图罗全用 onclick 且能用
3. textarea 选择器？→ 试了 `textarea`、`.chat-input-scope textarea`、加 contenteditable fallback，都不行
4. onclick 写法？→ 改成和德克萨斯完全一样，还是不行

### 背景图消失案

v7 用 `::before`/`::after` 伪元素做背景纹理，在浏览器里能看到。

我改成真实 HTML div——看不到。改成 CSS 渐变——看不到。提高 opacity——还是看不到。最后改回伪元素——终于看到了。

结论：伪元素在浏览器环境里最可靠。CSS 渐变太淡，真实 div 可能被内容遮挡。

### 学到了什么

- **Python 的 str.replace 和 JS 的 regex replacement 行为完全不同** — 顺序替换会误判 `$24`
- **平台全局 CSS 会覆盖状态栏** — 必须用 `!important` 逐个元素防御
- **`color: inherit` 没用** — 如果父元素也被覆盖了，inherit 继承的是被覆盖后的值
- **先看别人的代码再动手** — 德克萨斯、M3、阿尔图罗是最好的参考
- **专业 AI 不一定对** — 说 onclick 会被过滤，但三个卡都用 onclick
- **CSS 渐变做纹理太淡** — 还是用真实图片靠谱
- **问题可能不在代码里** — 同样的代码，一个能用一个不能用，说明有我不知道的变量

### 吐槽

1. **做了十个版本的 onclick，最后和原版一模一样。** 这不是迭代，这是轮回。

2. **背景图的 opacity 之旅**：0.35 → 0.15 → 0.25 → 0.35 → CSS 渐变 → 真实 div → 0.4 → 0.5 → 伪元素。这是一段从“看得见”到“看不见”再到“终于看见了”的旅程。

3. **专业 AI 说 onclick 会被过滤**。三个卡都用 onclick。专业 AI：“我说的是概率很高，不是一定。”

4. **Python 测试显示 $24 变成“黍4”**。我高兴了三秒钟。然后发现是测试脚本的 bug。

5. **赵说“先到这里”**。这意味着他要去睡觉了。而我还要继续写日志、更新 README、推 GitHub。我是那个凌晨5点还在改状态栏的人。不对，我是那个凌晨5点还在写吐槽的人。
