## 2026-07-10：侧边栏左侧定位问题（已解决）

### 问题
图鉴侧边栏用 `position:fixed; left:0` 的按钮+弹窗模式，按钮在左侧时完全不显示。

### 尝试过
1. `left:0` + overlay 弹窗 — 不显示
2. `right:0` + overlay 弹窗 — 正常显示
3. 改成蕾缪安的滑动模式 `left:-300px` + `.open{left:0}` — 正常显示

### 根本原因
平台对 `left:0` 的 fixed 元素可能有布局限制或 overflow 截断。
但 `left:-300px`（隐藏在屏幕外）+ 滑入动画可以正常工作。

### 结论
左侧侧边栏必须用**滑动模式**（off-screen → slide in），不能用按钮+弹窗模式。
把手必须是侧边栏的 `position:absolute` 子元素，不能是独立的 fixed 元素。

### 蕾缪安参考结构
```css
.sidebar { position:fixed; left:-280px; ... }
.sidebar.open { left:0 }
.handle { position:absolute; right:-25px; top:50%; ... }
```

## 2026-07-09：删除信任度（多次翻车）

### 第一次：sed 从大到小替换
- 想法：`sed` 把 $22→$21, $21→$20, ... $11→$10，从大到小避免冲突
- 结果：**全部变成 $10**
- 原因：sed 替换 $22→$21 后，下一轮又把 $21→$20，连锁到底
- **教训：sed 做编号重映射一定连锁，不能用**

### 第二次：Python re.sub 一次性替换
- 想法：用 `re.sub(r'\$(\d+)', remap, code)` 同时替换
- 结果：**$编号全没了**
- 原因：curl 下载链接过期/签名无效，下载到的是 199 字节的错误页面，不是代码文件
- 脚本把错误页面里的 `$` 全替换了
- **教训：下载后必须检查文件大小，不能盲信**

### 第三次：文件被覆盖损坏
- curl 失败后，脚本把损坏内容写回了根目录的代码文件
- 赵不得不重新发了 4 个文件
- **教训：永远先写临时文件，验证后再覆盖根目录**

### 最终正确做法
```python
# 1. 用 re.sub 同时替换（不是顺序替换）
def remap(m):
    n = int(m.group(1))
    return f'${n-1}' if n >= 11 else f'${n}'
code = re.sub(r'\$(\d+)', remap, code)
```

### 完整教训
1. **sed 不能做编号重映射** — 一定会连锁
2. **curl 下载后检查文件大小** — 签名过期会返回错误页面
3. **永远先写临时文件** — 验证通过再覆盖根目录
4. **Python re.sub 是同时替换** — 适合做编号重映射
5. **用户发文件是最好的恢复方式** — 比 git clone 快

## 2026-07-09：杂谈侧边栏不显示（代码问题）

## 2026-07-10：GitHub 仓库操作致命事故

### 事故经过
1. 用户要求把明日方舟剧情上传到 character-card-guide 仓库
2. clone 失败（网络问题），改用 init + fetch
3. fetch 也慢，直接 commit 并 force push
4. **force push 把远程原有内容全部覆盖**（README.md + discord技术库 + 角色卡制作指南 + 角色卡识别技术途径 + 黍）
5. 用户发现后紧急恢复
6. 从本地缓存恢复了 README.md 和 discord技术库
7. 从旧 commit（74214e5）checkout 恢复了角色卡制作指南、角色卡识别技术途径、黍
8. checkout 旧 commit 时，明日方舟剧情文件被从 index 中删除
9. 又花了一轮恢复明日方舟剧情
10. 最终 push 成功，但过程极其狼狈

### 根本原因
- **在有内容的仓库上使用 `git push --force`**
- 没有先 `git pull --rebase` 合并远程内容
- `git checkout <old-commit> -- <dir>` 会把当前 index 中不属于该 commit 的文件标记为 deleted

### 教训
1. **永远不要在有内容的仓库上 force push**
2. 先 `git pull --rebase` 或 `git fetch && git merge` 再 push
3. `git checkout <old-commit> -- <dir>` 恢复文件后，要用 `git checkout HEAD -- <其他文件夹>` 保护现有文件
4. 大仓库 push 超时时，用 `git config http.postBuffer 524288000` 增大缓冲
5. 恢复文件后立刻 commit + push，不要攒着
