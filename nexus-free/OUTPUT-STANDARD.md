# 角色卡输出标准

> 基于蕾缪安、黍两张卡的参考格式。
> 每个角色输出 3 个文件。

## 文件结构

```
[角色名]/
├── 人设-[角色名].txt        ← 角色人设（8000-10000字）
├── [角色名]世界书数据.json    ← 世界书（50-170条目）
└── [角色名]正则数据.json      ← 状态栏+开场白+正则脚本
```

## 一、人设文件（TXT）

### 章节结构
```xml
<名称>角色名</名称>
<性别>女</性别>
<外貌>详细外貌描述...</外貌>
<服装与装备>
【套装名称】常服
（整体风格）...
（上衣）...
（下装）...
（鞋履）...
（配饰）...
【武器】...

【其他服装】皮肤名
（整体风格）...
</服装与装备>
<基本设定>
简要概括
详细设定
评价
</基本设定>
<性格>
核心性格描述
喜好
癖好
不喜欢
性观念（DL向需要）
</性格>
<成长与经历>
早期经历
后续经历
</成长与经历>
<相关角色>
角色A：关系描述

</相关角色>
<台词>
【任命助理】台词
【交谈1】台词
...
</台词>
```

### 要求
- 字数：8000-10000字
- 台词必须来自官方来源（PRTS/BWIKI），严禁编造
- 使用 `{{char}}` 代表角色，`{{user}}` 代表玩家
- 服装描述精确到材质、颜色、纹样、剪裁

## 二、世界书文件（JSON）

### 格式
```json
{
  "entries": {
    "0": {
      "key": ["关键词1", "关键词2"],
      "keysecondary": [],
      "comment": "[分类]条目名",
      "content": "# 标题\n\n## 基本信息\n- 字段: 值\n\n## 详细内容...",
      "constant": false,
      "selective": true,
      "probability": 100,
      "position": 0,
      "order": 200,
      "uid": 0
    }
  }
}
```

### 条目分类
- `[设定]` — 角色核心设定（好感度逻辑、输出逻辑、页面美化）
- `[机制]` — 游戏机制（骰子系统、状态栏逻辑）
- `[角色]` — 相关角色条目（每个重要角色一个条目）
- `[地点]` — 相关地点
- `[组织]` — 相关组织
- `[世界观]` — 世界观设定

### 要求
- 条目数：50-170个
- 每个条目 content 至少 100字
- 关键词用于触发匹配
- 固定传输条目（constant=true）用于核心设定

## 三、正则数据文件（JSON）

### 格式
```json
{
  "pageDepth": 2,
  "statusbar": "《人物栏》【侧边栏】【侧边栏1】【侧边栏2】【侧边栏3】\n《游玩助手》\n《功能》《二次确认》《美化》《地图》",
  "beginning": "开场白全文（含状态栏）...",
  "regex_scripts": [
    {
      "scriptName": "脚本名",
      "findRegex": "正则表达式",
      "replaceString": "替换内容",
      "flags": "g",
      "markdownOnly": false,
      "disabled": false
    }
  ]
}
```

### 状态栏嵌入格式
开场白末尾嵌入状态栏数据：
```
【状态栏开始】
对话轮次：0
人物名：角色名
职阶：XX
状态：XX
情绪：正常
信赖度：80%
好感度：75%
内心独白：...
位置：XX
时间：XX
服装与装备：XX
【状态栏结束】
```

### 正则脚本用途
- 状态栏渲染（进度条、emoji替换）
- 侧边栏切换按钮
- 游玩助手功能
- 页面美化（CSS注入）
- 功能按钮（骰子、图鉴等）

## 数据源使用规则

1. **先全网搜索**：用 20 个数据源广泛收集信息
2. **再用方舟数据库兜底验证**：arknights_lore + arknights_story 确保准确性
3. **台词必须官方来源**：PRTS/BWIKI action=raw
4. **人物关系以设定库为准**：arknights_lore 的 `<相关角色>` 字段
