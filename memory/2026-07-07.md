### 第二轮：全局美化 CSS

赵发了截图，我看到了平台的全局美化 CSS：
```css
[class*="ai"] { color: var(--text-main) !important; }
[class*="bot"] { color: var(--text-main) !important; }
```

这个选择器匹配了状态栏所在的容器，用 `!important` 把所有文字颜色都覆盖了。

**第一反应**：用 `color: inherit !important` 让子元素从父元素继承颜色。
**结果**：没用。因为父元素也被覆盖了，inherit 继承的是被覆盖后的颜色。

**第二反应**：给每个元素单独写 `color: #xxx !important`。
**结果**：用了。写了 14 条规则，像打地鼠一样一个一个堵。

```css
.shu-sb .lb{color:#777!important}
.shu-sb .vl{color:#1a1a1a!important}
.shu-sb .shu-thought{color:#444!important}
.shu-sb .shu-time{color:#888!important}
.shu-sb .shu-round{color:#aaa!important}
.shu-sb .shu-bar-lb{color:#777!important}
.shu-sb .shu-bar-v{color:#5b8c5a!important}
.shu-sb .shu-loc{color:#777!important}
```

## 凌晨 05:30 - 07:00：按钮不工作之谜

### 问题

点击选项按钮后，textarea 没有填入对应句子。完全没反应。

### 排查路径

**第一轮：$24 歧义**

我怀疑 `$24` 被替换引擎解释成 `$2`（第2组）+ 字面量 `4`。写了个 Python 脚本测试：
```python
result = template
for i, g in enumerate(m.groups(), 1):
    result = result.replace(f'${i}', g)
```

结果：按钮内容变成了"黍4"、"黍6"、"黍8"。我立刻确认：就是 `$24` 的问题！

然后我把所有 `$24` 改成 `${24}`。赵试了，还是不行。

**但是**——我的 Python 测试脚本是**顺序替换**的。先替换 `$2`，把 `$24` 变成了 `黍4`，然后替换 `$24` 时已经找不到了。这是测试脚本的 bug，不是平台的问题。

用 regex 同时替换验证：
```python
result = re.sub(r'\$(\d+)', repl, template)
```

结果：`$24` 正确替换为"黍，生态舱的情况具体怎么样？"。所以 `$24` 没问题。

**教训：不要用 Python 的 `str.replace` 测试 JS 的 regex replacement。顺序替换和同时替换行为完全不同。**

**第二轮：onclick 被平台过滤？**

赵找了专业 AI 咨询。专业 AI 说：


**三个卡全部用 onclick。没有一个用 onerror。**
