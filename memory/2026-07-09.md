### 教训汇总（今日共 12 条）
1. sed 不能做编号重映射（连锁）
2. curl 下载后检查文件大小
3. 永远先写临时文件再覆盖根目录
4. Python re.sub 是同时替换
5. 用户发文件比 git clone 快
6. 不要假设工作区文件是最新的
7. 文件大小是线索（2KB 的“全局美化”不完整）
8. 一次只改一个变量
9. 平台可能过滤 HTML 注释
10. CSS 和 HTML 尽量紧凑
11. onclick 用 function(){} 不用箭头函数
12. 和能用的代码对比找差异比猜原因快
