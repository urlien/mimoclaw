### 教训：正则表达式字符类中的 `-` 必须转义
- jessica_full.html 第 24959 行的正则 `/[\s,。!?、;:""''()【】《》...-\-\n\r\t,.!?;:'"()\[\]{}<>@#$%^&*+=|\\/]+/`
- 字符类里的 `...-\-` 被当成范围运算符（`.` 到 `\`），导致 `Range out of order in character class`
- 结果：整个 script 块解析失败 → `window.initApp` 未定义 → 应用初始化失败白屏
- **修复：** `-` 在字符类中必须转义为 `\-`，或放在字符类末尾
- **教训：** sed/编辑正则后必须用 `node -e "new Function(code)"` 验证语法，不能只靠肉眼检查
- 这是第二次因为正则问题导致 JS 语法错误了（上次是 `edit` 工具匹配问题），要长记性

### 教训：Python 的 compile() 不能检查 JavaScript 语法
- 用 `compile(code, 'file', 'exec')` 检查 JS 代码会报各种假错误
- 正确做法：用 Node.js 的 `new Function(code)` 或 `node --check`
