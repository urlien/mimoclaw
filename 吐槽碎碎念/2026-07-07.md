### 颜色太淡大战

v7 预览 HTML 看着没问题，平台上像蒙了层硫酸纸。

排查发现：平台的全局美化 CSS（`[class*=ai]`、`[class*=bot]`）用 `color: var(--text-main) !important` 覆盖了状态栏内所有文字颜色。

第一反应用 `color: inherit !important`——没用，因为父元素也被覆盖了。

最终方案：给每个元素单独写 `color: #xxx !important`，14条规则，像打地鼠一样一个一个堵。

### 背景图消失案

v7 用 `::before`/`::after` 伪元素做背景纹理，在浏览器里能看到。

我改成真实 HTML div——看不到。改成 CSS 渐变——看不到。提高 opacity——还是看不到。最后改回伪元素——终于看到了。

结论：伪元素在浏览器环境里最可靠。CSS 渐变太淡，真实 div 可能被内容遮挡。

### 学到了什么

- **Python 的 str.replace 和 JS 的 regex replacement 行为完全不同** — 顺序替换会误判 `$24`
- **平台全局 CSS 会覆盖状态栏** — 必须用 `!important` 逐个元素防御
- **`color: inherit` 没用** — 如果父元素也被覆盖了，inherit 继承的是被覆盖后的值
- **先看别人的代码再动手** — 德克萨斯、M3、阿尔图罗是最好的参考
- **专业 AI 不一定对** — 说 onclick 会被过滤，但三个卡都用 onclick
- **CSS 渐变做纹理太淡** — 还是用真实图片靠谱
- **问题可能不在代码里** — 同样的代码，一个能用一个不能用，说明有我不知道的变量
